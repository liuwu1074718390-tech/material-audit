# 更简单的部署方案

## 🎯 方案对比

### 方案一：使用 Vercel 部署（最简单，推荐！）

**优点：**
- ✅ 完全免费
- ✅ 自动 HTTPS
- ✅ 全球 CDN 加速
- ✅ 自动部署（Git Push 即部署）
- ✅ 无需配置服务器
- ✅ 支持 Nuxt 3

**步骤：**

1. **准备代码**
   ```bash
   # 在本地电脑，确保代码已提交到 GitHub
   git add .
   git commit -m "准备部署"
   git push
   ```

2. **访问 Vercel**
   - 打开：https://vercel.com
   - 使用 GitHub 账号登录

3. **导入项目**
   - 点击 "New Project"
   - 选择你的代码仓库
   - 框架预设选择 "Nuxt.js"

4. **配置环境变量**
   在项目设置中添加：
   ```
   DIFY_API_KEY=你的密钥
   DIFY_API_URL=你的地址
   DB_HOST=数据库地址
   DB_PORT=3306
   DB_USER=数据库用户
   DB_PASSWORD=数据库密码
   DB_NAME=数据库名称
   ```

5. **部署**
   - 点击 "Deploy"
   - 等待 2-3 分钟
   - 获得链接：`https://your-project.vercel.app`

**完成！** 就这么简单！

### 方案二：使用 Railway 部署（支持 Docker）

**优点：**
- ✅ 简单易用
- ✅ 支持 Docker
- ✅ 自动 HTTPS
- ✅ 免费额度

**步骤：**

1. **访问 Railway**
   - 打开：https://railway.app
   - 使用 GitHub 账号登录

2. **创建项目**
   - 点击 "New Project"
   - 选择 "Deploy from GitHub repo"
   - 选择你的仓库

3. **配置环境变量**
   - 在项目设置中添加所有环境变量

4. **部署**
   - Railway 会自动检测并部署
   - 获得链接：`https://your-project.up.railway.app`

---

### 方案三：使用腾讯云轻量应用服务器（Linux，更简单）

如果你可以重新购买或更换服务器，建议使用 **Linux 系统**的轻量应用服务器：

**优点：**
- ✅ 使用我们提供的自动化脚本，一键部署
- ✅ Linux 系统更适合部署 Node.js 应用
- ✅ 命令更简单

**步骤：**

1. **购买轻量应用服务器**
   - 在腾讯云选择 **Ubuntu 20.04** 或 **CentOS 7**
   - 配置：2核2GB 即可

2. **使用自动化脚本部署**
   ```bash
   # 连接服务器
   ssh root@新服务器IP
   
   # 运行部署脚本
   bash scripts/deploy-tencent.sh
   ```

3. **完成！**

### 方案四：在 Windows 服务器上使用 Docker（如果支持）

如果 Windows 服务器支持 Docker：

1. **安装 Docker Desktop for Windows**
   - 下载：https://www.docker.com/products/docker-desktop

2. **使用 Docker 部署**
   ```powershell
   # 进入项目目录
   cd C:\material-audit
   
   # 构建镜像
   docker build -t material-audit .
   
   # 运行容器
   docker run -d -p 3000:3000 --name material-audit --env-file .env material-audit
   ```

### 方案五：使用腾讯云 Serverless（无需服务器）

**优点：**
- ✅ 无需管理服务器
- ✅ 按量付费
- ✅ 自动扩缩容

**步骤：**

1. **安装 Serverless Framework**
   ```bash
   npm install -g serverless
   ```

2. **创建配置文件**
   在项目根目录创建 `serverless.yml`：
   ```yaml
   component: nuxt
   name: material-audit
   
   inputs:
     src: ./
     region: ap-guangzhou
     runtime: Nodejs18.15
   ```

3. **部署**
   ```bash
   serverless deploy
   ```

## 💡 我的推荐

### 如果你想要最简单的方式：
**使用 Vercel** - 5 分钟完成部署，完全免费，自动 HTTPS

### 如果你必须使用腾讯云服务器：
1. **首选**：购买 Linux 系统的轻量应用服务器，使用自动化脚本
2. **次选**：继续使用 Windows 服务器，但需要手动配置（参考 Windows 部署指南）

### 如果你想要更多控制：
**使用 Railway** - 简单但功能强大

---

## 🚀 快速决策

**问自己：**
- ❓ 是否必须使用腾讯云 Windows 服务器？
  - 是 → 使用 Windows 部署指南
  - 否 → 推荐使用 Vercel 或购买 Linux 服务器

- ❓ 是否希望完全自动化？
  - 是 → 使用 Vercel 或 Railway
  - 否 → 可以手动配置

- ❓ 是否需要更多服务器控制？
  - 是 → 使用 Linux 服务器 + 自动化脚本
  - 否 → 使用 Vercel
